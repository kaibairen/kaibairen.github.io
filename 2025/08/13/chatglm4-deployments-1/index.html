启动阶段

Step 1 创建部署ChatGLM4需要的实例

   首先，我们需要创建部署大模型（以GLM-4-9B为例）所需要的实例。登录华为云个人开发平台后按照在华为云创建Notebook工作实例给出的路径 主菜单 → 人工智能 → AI开发平台ModelArts → 开发空间 → Notebook → 创建/创建Notebook 打开创建实例界面。
   由于本次的任务并不涉及模型训练，因此在算力选择上仅使用基础配置即可(Snt9单卡)，具体可参考设置如下：

Step 2 初步了解Jupyter Lab可视化操作界面

   华为云AI开发平台ModelArts为开发者提供了多样化的方式访问Notebook，保证开发者在过程中按照个人喜好选择合适的开发方式。我们将以云上Jupyter Lab为例，在实例上部署GLM-4-9B模型。
   我们将会介绍访问华为云Notebook的云上Jupyter Lab可视化操作界面。

Step 4 配置部署GLM-4-9B的环境

创建新的虚拟环境 glm4-9b

   我们可以通过输入conda deactivate退出已打开的虚拟环境，并使用conda create -n glm4-9b python=3.8创建部署LLM使用的虚拟环境。

Tips1： 在运行过程中我们偶尔会遇到CondaHTTPError指出在设置的镜像源中无法找到需要的包（如下）我们可以通过conda config --remove-key channels清除所有的自定义通道，并重新添加清华源:
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
conda config --set show_channel_urls yes
重新创建虚拟环境，发现问题解决。

   虚拟环境创建成功后，使用conda activate glm4-9b即可激活新环境glm4-9b。

配置Jupyter Notebook 的内核（Kernel）

   新建一个test.ipython的notebook文件，打开后点击选择Kernel发现并不存在我们新创建的虚拟环境的选项。根本原因是我们还未将其作为新的内核添加。

安装ipykernel

   首先，在当前环境（glm4-9b）键入pip install ipykernel安装ipykernel。

查看版本/验证安装

   安装完成后，可以通过python -m ipykernel --version查看版本，出现以下内容说明安装成功。

添加内核

   在此基础上，输入python -m ipykernel install --user --name=内核名 --display-name 显示的内核名 添加内核，内核真名不必要与环境名相同，但是与环境一致的内核名有利于对齐。添加成功后内核就会指向当前环境。

删除无用图标

   最后，输入rm -r /home/ma-user/.local/share/jupyter/kernels/glm4-9b/logo-*可以删除内核名为glm4-9b的无用图标，至此kernel设置完成。同时在Lancher界面可以发现自定义的虚拟环境Kernel的图标。

其他操作

   在使用完成后，如果有需要我们可以使用jupyter kernelspec uninstall my-py3-tensorflow-env删除新添加的Kernel。

下载运行必须的python package

   我们可以通过访问Huggingface了解部署GLM-4-9B需要的环境，官方最新更新的文档建议从transformers>=4.46.0开始，减少后续更新可能面临的兼容性问题。打开创建的虚拟环境，使用命令行pip install transformers>=4.46.0下载需要transformers库。我们可以通过pip show transformers（Terminal）或!pip show transformers（Notebook）查看是否下载成功（如下）。

Tips2： 在下载transformers时偶尔会出现报错，原因是缺少必要的package，只需要按照报错pip缺少的内容即可。
