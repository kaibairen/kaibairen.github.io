数组和链表

需要存储多项数据时，存在两种基本方式——数组和链表。

需要明确的是数组和链表并非都适用于所有的情形，因此我们需要知道他们的差别&优缺点。

数组

使用数组意味着所有的元素在内存中都是相连的（需要分配连续的内存空间）。

在数组中添加新元素时，若当前连续空间后的内存已被占用，则需要重新分配连续的内存。

我们可以通过预留空间的方式避免多次重新分配内存空间（即当前虽然只有3个元素，但是创建大小为10的数组），但是这种方法也面临以下问题：

  • 1、预留的空间可能不被充分使用，导致内存浪费
  • 2、当元素大于10时，数组仍需要被重新分配空间

链表

链表中的元素可以存在于内存的任何地方，通过在链表的元素中存入指向下一元素的地址，从而将一系列随机的内存地址串联到一起。

常见操作的复杂度

数组 链表
读取 O(1) O(n)
插入 O(n) O(1)
删除 O(n) O(1)
  • 不同于插入操作，删除操作总能成功。这是因为插入操作收到内存空间大小影响，而删除操作空间永远是足够的。
