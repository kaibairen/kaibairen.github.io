1.1 对单一任务进行微调

灾难性遗忘（Catastrophic Forgetting）

现象简述

原因： 全量微调过程中改变LLM原始的模型权重
表现：尽管可能在微调目标的单一任务上表现良好，但是会降低在其他任务上的性能

解决方法

  1. 确定灾难性遗忘是否真的会影响到你的使用场景

    在只针对特定任务场景的情况下，模型的泛化能力并不被需要，则不会造成什么影响

  2. 同时在多个任务下进行微调
  3. 执行参数有效微调 Parameter Efficient Fine-tuning（PEFT）

    PEFT保留了模型的原始权重，并且只训练少量特定任务的适配器层（adapter layers）和参数

1.2 Evaluation Metric

ROUGE（召回率导向的摘要评估）

ROUGE主要用于通过将自动生成的摘要和人工生成的参考摘要进行比较来评估生成文本的质量

Used for text summarization
compares a summary to one or more reference summaries


在了解ROUGE的公式前，我们需要了解语言解剖学中的相关信息gram。如上图所示，在句子中单个单词被称为unigram,两个连续单词被称为bigram，当连续的单词数>=2时统一被称作n-gram

ROUGE-1

简介：ROUGE-1仅关注生成文本的单个单词，而不关注单词的顺序。1指代的是计算过程中使用单个单词（unigram）作为衡量的依据进行计算。

  • Recall（召回率）

  • Precision（精确度）

  • F1
    F1 score的计算方法是Precision和Recall方法的调和平均值

Tips: 缺陷：易产生误导，生成得分高但是主观上质量差的句子。
Example：
Reference：It is cold outside.
Generation output 1: It is not cold outside.
Generation output 1: It is very cold outside.
我们不难通过计算发现，ROUGE-1计算出的score是相同的

ROUGE-2

简介：ROUGE-2 2指代的是计算过程中使用两个单词（bigram）作为衡量的依据进行计算。通过考虑bigram或者参考句子和生成句子的单子对的集合，可以作为一种极其直观的方式理解句子中词语的排列顺序，缓解ROUGE-1的缺陷并得到相对公正的分数。

  • Recall（召回率）

  • Precision（精确度）

  • F1

ROUGE-L

简介：相比于举一反三的使用ROUGE-n通过n-gram作为衡量依据进行计算，不如采用更有效的方法。 在ROUGE-L方法中，我们将通过在生成的句子和参考句子中存在的最长公共子序列(LCS)作为计算的依据。

  • Recall（召回率）

  • Precision（精确度）

  • F1

相关提示

  1. 不同任务中的ROUGE分数是无法相互比较的
  2. 简单的ROUGE分数的一个特别问题是一个坏的生成结果可能会对应一个极好的分数，如下图：

BLEU（双语评估）

BLEU主要用于通过将自动生成的翻译和人工生成的参考翻译进行比较来评估生成文本的质量

Used for text translation
Compares to human-generated translations
BLUE分数本身是通过对多个n-gram大小的平均精确度进行计算得出

当生成句和参考句越接近，BLEU的数值就越接近1

总结

  1. BLEU和ROUGE都是相对简单的指标，计算成本较低，在模型迭代过程中可用于简单参考
  2. 但在大模型整体性能的最终评估阶段，不能只参考这两种评估基准。我们还需要参考其他由研究人员开发的评估基准（benchmarks）
